# Идея взаимодействия компонентов основной сессии

### У пользователя два основных маршрута которые он может пройти:

**Из основного меню:**
```
Пользователь -> Main -> RegForm -> Session -> <Игра /> -> <Field /> -> ...
```

**Подключиться по ссылке:**
```
Пользователь -> Session -> RegForm -> <Игра /> -> <Field /> -> ...
```


Пока что хочу сконцентрироваться просто на логике веб-сокета, чтобы потом вынести его отдельно, и буду делать всё внутри TicTacToe, чтобы потом уже понять где зона ответственности сессии, где оркестратора игры, а где уже основной логики.

**Идея такая:**
```html
    <TicTacToe/>
            ↓   ↑   - (1)
            <Field/>
                ↓   ↑   - (2)
                <Cell/>
```

**Передача первого уровня TicTacToe - Field:**
TicTacToe -> Отправляет через props ws_data, со всем что может понадобиться полю, например кто сейчас ходит, или состояние поля
-> Field -> Принимает данные от оркестратора игры, и например переводит в нужный для себя вид, например объединяет всё поле в один массив
